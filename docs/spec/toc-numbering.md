# 目次と特殊章の採番 要求・要件

## 概要

目次（`nav[role="doc-toc"]`）と本文の見出しに対する自動採番の要求・要件を
定める．通常の章の連番から除外する章種別の一覧と，付録・解答の専用採番，
および新しい特殊章種別を追加する際の拡張方法を明文化する．

## 目次

- 背景と要求（why）
- 用語集（ユビキタス言語）
- 要件（what）
- 拡張方法
- スコープ外
- 変更記録

## 背景と要求（why）

本テンプレートの目次は，章項目へ CSS カウンターで連番を振る．
まえがきや奥付など番号が不要な章は，カウンターの対象から除外する
（[Issue #10](https://github.com/tomio2480/techbook-template/issues/10)，
[PR #12](https://github.com/tomio2480/techbook-template/pull/12)）．

派生書籍リポジトリ
[techbook-introduction-to-electronics-basic-led](https://github.com/tomio2480/techbook-introduction-to-electronics-basic-led)
では，この仕組みを解答章と付録章へ拡張した実績がある．
解答や付録を持つ書籍構成は一般にありうるため，テンプレート側へ一般化して
取り込み，派生書籍で再利用できる状態を保つ
（[Issue #27](https://github.com/tomio2480/techbook-template/issues/27)）．

価値提案は次のとおりである．執筆者が frontmatter の `class` 指定と
ファイル命名だけで，特殊章の採番（本文・目次の両方）を追加設定なしに
得られる状態を維持する．

## 用語集（ユビキタス言語）

表 1. 本文書で使う用語の定義．

| 用語 | 意味 |
|---|---|
| 特殊章 | 通常の章番号連番の対象から外す章．まえがき・付録・解答など |
| 除外 | 目次の章カウンター（`toc-chapter`）を増やさない扱い |
| 専用採番 | 通常連番の代わりに適用する採番．付録の `A.` や解答の `A.n.` など |
| href 部分一致 | 目次項目の `a[href*="..."]` により章種別を判定する方式 |

## 要件（what）

### 除外対象の章種別

目次の章カウンターから除外する章種別を表 2 に示す．
判定は目次項目の href 部分一致で行う．

表 2. 目次の章番号カウントから除外する章種別の一覧．

| 章種別 | href の一致文字列 | 目次での採番 |
|---|---|---|
| 表紙 | `cover` | なし |
| まえがき | `preface` | なし |
| 目次自体 | `toc` | なし |
| 解答 | `answer` | `A.`／`A.n.` |
| 付録 | `appendix` | `A.`／`A.n.`／`A.n.n.`（出現順に A，B，…） |
| あとがき | `afterword` | なし |
| 奥付 | `colophon` | なし |

### 付録の採番

- 本文は `class: appendix` と `counter-set: chapter N` の組で章番号を
  アルファベット表示する（[Issue #8](https://github.com/tomio2480/techbook-template/issues/8)）．
- 目次は付録の出現順に A，B，… を割り当てる．
  節・項は `A.1.`／`A.1.1.` の形式とする．
- 受け入れ条件: 既定のサンプル構成で付録の目次項目が `A.` になり，
  通常章の連番（1〜3）へ影響しないこと．

### 解答の採番

- 本文は `class: answers` の指定で章見出しに `A.` を付け，
  節見出しを `A.1.` 形式で採番する．
- 節番号は対応する章の番号と一致させる想定である．
  第 1 章の解答を最初の節に置くと `A.1.` になる．
- 目次も同じ形式（`A.`／`A.n.`）で採番する．
- 受け入れ条件: 解答章を entry へ追加したとき，目次と本文の採番が
  一致し，通常章の連番へ影響しないこと．

### 既知の制約

- href 部分一致方式のため，一致文字列を含むファイル名の通常章は
  誤って除外される．通常章のファイル名に表 2 の文字列を含めない．
- 付録の目次番号は目次内の出現順で決まる．本文側で `counter-set` の
  開始値を変えた場合，目次側の文字と一致しない．
  一致させるにはカスタム CSS で目次側の上書きが必要である．
- 解答の目次採番は 2 階層（`A.`／`A.n.`）までとする．
- 解答の `A.` と最初の付録の `A.` は接頭辞が衝突する．
  併用する書籍では付録の文字変更などの調整を検討する．
  参考実装の書籍は付録を `X.` に変更して回避した．

## 拡張方法

新しい特殊章種別を追加する場合，`config/themes/techbook/theme.css` の
目次セクションで次の 2 点を編集する．

1. 除外リストへの追加．`:not(:has(...))` と `:has(...)` の各セレクタに
   `> a[href*="<一致文字列>"]` を加える．対象は章カウンター適用・
   子セクション抑止・節・項の全ルールである．
2. 専用採番ルールの追加．子セクション抑止ルールより後ろに，
   同じ詳細度のセレクタで `::before` の content を上書きする．
   記述順により後勝ちで適用される．

本文側の採番が必要な場合は，`.answers` のルール群を雛形として
章種別のクラスを新設する．

## スコープ外

- 一致文字列を設定ファイルで変更できる仕組み．既存 5 種と同じ
  ハードコード方式を踏襲し，設定機構は必要が生じるまで作らない．
- 解答章内の図・表・数式の専用番号形式．解答章の想定用途では
  出現頻度が低いため，必要になった時点で検討する．
- 解答の 3 階層目（`A.n.n.`）以降の採番．
