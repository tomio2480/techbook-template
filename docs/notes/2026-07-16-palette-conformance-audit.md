# 配色パレット準拠の全数点検と追従範囲の確定

Issue #29 / PR #30 の決定記録である．テーマ CSS の色指定を全数点検し，
基調色へ追従させる要素と，意図的に追従させない要素の線引きを確定した．
検証手法（computed style 照合）とレビューで得た落とし穴も残す．

## 背景

Issue #22 で配色を 2 層トークン化したが，追従漏れが残っていた．
章扉「この章で学ぶこと」枠と表への準拠要望を機に全数点検を実施し，
柱・リンク・引用を含む計 5 要素の漏れを確定して一括対応した．

## 追従させた要素

対応した要素を表 1 に示す．いずれも意味トークンを新設または変更し，
`theme.css` は意味トークンのみを参照する規約を維持した．

表 1．基調色へ追従させた要素と対応内容．

| 要素 | 対応 |
|---|---|
| 章扉トピック枠 | `--chapter-topics-bg` と `--chapter-topics-accent` を新設 |
| 表 | `--table-head-rule` ・ `--table-rule` ・ `--table-stripe-bg` を新設 |
| 柱（章名ヘッダー） | `--running-head-color` を新設 |
| リンク | `--color-link` を `--palette-primary` 参照へ変更 |
| 引用 | `--quote-bg` と `--quote-border` を新設 |

判断基準は次の 2 点である．第 1 に，柱とリンクはパレット差し替え後も
青系が残る唯一の箇所となり，不統一が目立つ．第 2 に，引用がコラムと
同系色になっても，種別判別を色に依存させない方針により問題がない．

## 追従させない要素の基準

- コードブロックの背景・枠・行番号帯．`code-highlight.css` の
  シンタックス色は，背景 #f5f5f5 前提でコントラスト比 4.5:1 を確保
  している．背景を変えると保証が崩れるため無彩色を維持する．
  この結合関係は `palette.css` の冒頭コメントへ明文化した．
- 奥付．装飾しない決定（装飾デザイン決定記録）に従う．
- `#fff` リテラルと `.sample-label` ．非追従の方針が明文化済みである．
- 水平線 `hr` と脚注罫．中立の区切り罫として維持する．

## 代替案と棄却理由

- 表の本体罫線をグレー維持する案．基調色 mid（#b0c4de）は従来の
  #999 より薄い．見出し罫を濃色にし，ゼブラで行の判別が残るため
  追従案を採り，PR で濃さの印象確認を依頼して確定した．
- `--palette-link` 独立スロットを維持する案．消費者が 1 箇所のみで
  基調色と役割が重複するため，スロットを削除し直参照へ集約した．

## 実装上の学び

- 表の直前段落には `theme.css` が採番接頭辞 `表N.-M: ` を自動付与
  する．原稿へ手書きした `表:` 接頭辞は二重表記になる（レビューで
  検出）．
- 印刷テーマの長さ単位は pt へ統一する．4px = 3pt の等値変換により
  見た目を変えずに置換できる．
- 意味トークン層に消費者のいないトークンを残さない．参照が消えた
  `--color-surface-faint` は削除した．
- 装飾カタログにはパレット追従要素をすべて載せる．見本の無い要素は
  差し替え時に確認できない．今回，表・引用・リンクの見本を追加した．

## 検証手法の学び

本環境では PDF のページ描画を直接確認できなかった．`pdftoppm` が
未導入で，内蔵ブラウザは `file://` と PDF 表示を拒否し，スクリーン
ショットもタイムアウトした．代替手順を次に示す．

1. `npx http-server` でリポジトリを配信し，生成 HTML を開く．
2. `getComputedStyle` で対象要素の色を取得し，期待値と数値照合する．
3. 再ビルド後はクエリ文字列（?v=N）を付け，ブラウザキャッシュの
   旧 HTML を回避する．

色の確認は目視より数値照合が確実である．ただし柱のようにページ
組版でのみ現れる要素は HTML 描画で確認できず，PDF の確認を
ユーザーへ依頼する形で補った．
