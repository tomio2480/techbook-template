# 枠アイコン本採用と Caution 改名の知見

Tips・注釈・注意の 3 枠を基調色へ統一した PR #37 に関する記録である．

運用の学びを記す．

## 背景

- 枠 3 種は第 1 層で基調色と独立した 3 系統（緑・黄・赤）だった．
  基調色パレットを差し替えても，枠だけ旧配色が残る点は課題だった．
- 配色を基調色へ統一すると色による種別判別が消える．
  代わりの判別手段が必要になった．

## 判断

- Issue #22 と同じ 2 段階方式を再適用した．見本ビルドに 3 案を
  並置し，目視フィードバックで 1 案へ集約してから本採用した．
  濃度（0.08 → 0.12）・記号廃止・帯の英語化も対話で確定した．
- アイコンはデータ URI の SVG 線画を `mask` で塗る方式とした．
  データ URI 内では CSS 変数を使えない．そこで色は
  `background-color` に持たせ，意味トークン経由でパレットへ
  追従させた．集中線は SVG 内へ組み込み，擬似要素 1 個で完結させた．
- 「注意」の英訳は Caution とした．`ANSI Z535` 対訳だと
  Caution ＝注意であり，軽い注意喚起という用途の温度感に合う．
  紙面表記だけでなくクラス名・トークン名も `caution` 系へ
  一括改名した．制作中の書籍は 1 冊だけで，改名するなら今だった．
- 種別記号（◆・※・！）は廃止した．記号ゆえに警戒していた
  `JIS X 0208` 制約（フォント差の豆腐化）は考慮自体が不要になった．
- 参照が消えた第 1 層の旧 3 系統（accent / note / warning）は
  廃止した．差し替え口を残さない判断は YAGNI に沿う．

## 代替案と棄却理由

- 角ワンポイント・帯先頭置換の 2 案は見本比較で棄却した．
  依頼者イメージは「薄色の大判を背景へ敷く遊び心」だった．
- 英訳 Warning は棄却した．実装語彙（`.warning`）との一貫性はある．
  しかし警告という強い温度感は用途に合わない．
  語彙の一貫性は改名で確保する方針へ倒した．
- `gemini-code-assist` 提案の `--recap-*` 参照は棄却した．章まとめ用トークンへ
  枠 3 種を結合させると，意味トークンの独立性が崩れる．

## 運用の学び

- 第 2 層トークンの定義を一時的に別ファイルへ仮置きする場合，
  第 1 層参照が規律違反に見える．定義位置の仮移設だという許容根拠を
  コメントへ明記しないと，Bot レビューで指摘される．
- `overflow: hidden` を枠へ足すと，枠幅からはみ出す図表が
  切り取られる．制約は theme.css のコメントへ明文化した．
- スタックド PR だった #35 は，親 #34 のマージ前に base を main へ
  付け替えた．ただし base 変更では CI が再実行されない．
  close/reopen は権限外だったため，一時 worktree のローカルテストで
  代替した．
- 別セッションの worktree が残ると，ブランチ削除やディレクトリ削除は
  ロックで失敗する．`git worktree prune` で登録は消せる．
  実体の削除はプロセス終了後になる．
