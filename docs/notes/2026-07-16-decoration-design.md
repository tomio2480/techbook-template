# 書籍要素の装飾デザイン決定記録

書籍要素（章扉・見出し・枠もの）の装飾強化に関する決定記録である．
バリアント見本方式による選定の結果と，設計判断・不採用案を残す．
関連は Issue #22・PR #23（見本）・フェーズ 2 PR（本採用）である．

## 背景

テンプレートの装飾は最小限であり，商業技術書らしいあしらいが不足していた．
実教出版・コロナ社・オーム社のような工業系教科書を目標イメージとした．
淡色基調で本文の邪魔をしないが，手の込んだ装飾を感じさせる方向である．

## 進め方の判断

Claude Design への外注は不採用とした．対象が Vivliostyle の組版 CSS で
あり，改ページ・counter・PDF タグ付けの検証は実ビルドでしか行えない．
代わりにバリアント見本方式を採った．要素ごとに 3 案を見本 PDF に並置し，
ユーザーが実物を見て選定する 2 フェーズ構成である．

フェーズ間に人間の選定ゲートが挟まるため，PR は 2 本へ分割した．
見本 PR は `Refs #22`，本採用 PR は `Closes #22` とし，クローズ担当を
本採用側に一本化した．

## 設計判断

### 配色の 2 層トークン化

配色は `palette.css` の 2 層 CSS カスタムプロパティへ外出しした．
第 1 層（パレット層）は本ごとに差し替える生の色である．
第 2 層（意味トークン層）は装飾要素が参照する意味名である．
theme.css は意味トークンのみを参照し，パレット層へ直接触れない．
これにより配色替えはパレット層の書き換えだけで完結する．

白（#fff）だけは意図的にリテラル運用とした．紙面の地色であり，
グラデーションのフェード先・白抜き文字も紙色と同一だからである．

### 記号は JIS X 0208 内に限定

種別記号は ◆・※・！ とし，JIS X 0208 内へ限定した．
⚠（U+26A0）は環境により絵文字表示になる恐れがある．
✎・✓ は JIS 外であり，ビルド環境のフォント差で豆腐化しうる．
フォント埋め込み自体は pdffonts で全 emb=yes を確認済みであり，
実リスクは印刷所でなく CI 等のビルド環境のフォント差である．
チェック印は罫線描画（回転 L 字）で代替し，フォント依存を消した．

### PDF タグ付けへの配慮

タグ付け後処理（OpenDataLoader）はレイアウト解析ベースである．
装飾追加にあたり次を守った．

- 本文テキストの絶対配置を増やさない．絶対配置は空要素の装飾のみとする．
- 見出しサイズの単調減少（h1 > h2 > h3 > 本文）を維持する．
- 枠内タイトルは 10pt 前後にとどめ，h3 と紛れさせない．
- `::before` の生成テキストは短く意味のある語のみとする．

### グレースケール判別

色相差へ必ず明度差を重ね，種別判別は記号・ラベル・罫線で担保した．
パレット差し替え時の制約（濃・中・淡の明度役割）は `palette.css` の
冒頭コメントに明記した．

## 不採用案と棄却理由

- 章扉 A 案（グラデ帯「第 N 章」）・B 案（コーナー色面）: 採用 C 案より
  重く，面を塗らない軽やかさを優先した．
- h1 B 案（縁取り特大番号）・C 案（番号プレート大版）: 扉 C 案の縁取り
  数字と役割が重複する．
- h2 A 案（グラデ帯）・C 案（縁取り番号）: 節が頻出するため，帯の面積や
  白抜き番号の主張が強すぎる．
- h3 B 案（小プレート）・C 案（左短罫＋下点線）: 項は最軽量の装飾が
  適切と判断した．
- まとめ A 案（ドッグイヤー）・B 案（角丸枠）: チェックリストの視覚的な
  「振り返り」性を優先した．
- コラム B 案（上部グラデ地）・C 案（上下罫線）: ドッグイヤーの遊び心を
  コラムの識別子として採用した．
- Tips 等 A 案（左罫線 3 段階）・C 案（ドッグイヤー 3 色）: 種別記号入りの
  帯が最も判別しやすい．

## 実装上の学び

- グラデーションの `transparent` 終端は透明黒を経由して黒ずむ．
  紙面が白前提のため，白へのフェードで実装する．
- `break-before` / `break-after` はブロックレベルボックスにのみ効く．
  inline-block のラベルには効かないため display を切り替える．
- 角落とし（ドッグイヤー）は背景の斜めハードストップグラデで作れる．
  折り返しの三角形は空の `::after` の絶対配置で足す．
- `.tips-title` は `p` 要素のため，`.tips p` のセレクタが勝ちうる．
  本文段落側は `:not(.tips-title)` で除外して汚染を防ぐ．
- 一括検証として，カスタムプロパティの定義・参照突合を機械で行うと
  未定義参照と未使用トークンを同時に検出できる．
